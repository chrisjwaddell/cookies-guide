Every network request


A very unique thing about cookies is that they are sent automatically. You don't have to do anything to send them, they just get added automatically with every network request. This is an advantage in some situations.

HTTP is stateless. Cookies make a request stateful. That's why they were integrated into HTTP.

Cookies are just headers with data in it. The cookies header is just like other headers like Authentication or Content-Length. You can send cookies in the Authentication header, you just need to do it manually. But cookies are sent automatically when sending cookies in the cookies header. When a request is made, whether it's a GET request for a webpage or a fetch request, the browser automatically adds the document.cookies string to the cookies header for every request.

Some cookie attributes such as domain, path and other cookie security attributes show or hide cookies for certain pages.

Fetch has extra security headers added to the request so to send a cookie via fetch, you need the 'credentials' property set to 'include'. You also need to allow some header settings on the server.
