Attributes - secure

With this setting, the cookie only sends for secure HTTPS websites. This stops man-in-the-middle attacks.

You should always try to make your cookies inaccessible on the client-side using JavaScript.

Use it like this:
document.cookie = “name=Batman; secure”

Insecure sites (with http: in the URL) can't set cookies with the Secure attribute.

But, don't assume that Secure prevents all access to sensitive information in cookies. For example, someone with access to the client's hard disk (or JavaScript if the HttpOnly attribute isn't set) can read and modify the information.
